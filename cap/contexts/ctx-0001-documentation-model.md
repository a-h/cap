# Documentation model

## Description

The standard for documenting a system: the entities an author writes as Markdown
files, the identifiers and references that connect them, and the on-disk layout that
holds them. This context defines what a well-formed model is, and the capabilities
that read and write it. It is concerned with the shape of the documentation, not with
the reports produced from it, which belong to the Reporting context (ctx-0002).
